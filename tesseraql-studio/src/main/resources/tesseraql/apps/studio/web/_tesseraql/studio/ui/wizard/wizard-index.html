Setup wizards

Each wizard generates a config fragment to merge into config/tesseraql.yml — nothing is changed in place. Set up an identity realm first (it backs who can sign in), then add the protocol your identity provider speaks.

Identity realm mapping →

Where user accounts live (a managed store or your own SQL). Start here.

SAML SP →

Federate logins with a SAML 2.0 IdP (Okta, ADFS, Azure AD).

OIDC provider →

Federate logins with an OpenID Connect provider (Google, Entra ID, Auth0).

SCIM provisioning →

Sync users and groups in or out over SCIM 2.0.
