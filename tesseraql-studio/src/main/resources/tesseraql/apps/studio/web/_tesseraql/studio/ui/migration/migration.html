New migration

Creates a Flyway migration under db/…/migration. A versioned migration is auto-numbered (next: V1 for the main datasource); a repeatable one is R__<name>. The new file needs a restart + migrate to be applied — the running app only lists it. Flyway has no built-in undo on the free edition: roll back by writing a follow-up migration (fix-forward).

From schema changes

Generate a migration from the difference between the captured schema baseline and the current schema — to capture changes made directly to the database back into a migration. Additive changes become real DDL; destructive ones are emitted as comments to review.

No schema baseline captured. Copy .tesseraql/docs/schema.json to schema.baseline.json (e.g. at a release), then this generates the migration for changes since.

DDL builder (optional)

Generate standard DDL for a common operation and drop it into the DDL field above to review and refine. Identifiers are not validated — check the result.
