Copilot

The copilot is not configured.

TesseraQL ships no model. Point tesseraql.copilot.endpoint at an OpenAI-compatible chat-completions URL, name a model, set enabled: true, and keep the key a secret reference (apiKey: ${secret.env.COPILOT_KEY}). See docs/copilot.md.

The copilot drives the same gated loop you do: it reads routes, sources, lint and schema, previews buffers, and — only with your edit role — saves drafts. Nothing it does is served until you review and apply the draft in the editor; every saved draft lands in the audit trail under your name.

You hold no Studio edit role, so the copilot answers questions but cannot save drafts in this session.
